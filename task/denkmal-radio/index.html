This task is about creating a concept for adding an "Internet Radio" to Denkmal.org. With this task we'd like to see a candidate's ability to design a tiny application, and how to integrate it in a larger context.

Description

Denkmal.org is a an open-source side-project part of the company is contributing to. It's a Basel-based event-calendar with the ability to pre-listen to an event's music. The application follows the mobile-first philosophy. We now would like to add a simple Radio to the app.

Requirements

  • We need an indication (button, link, etc.), on the landing page, that there is a radio
  • A radio interface with a playlist (You're only able to skip, or go to a certain song)
  • Think flexible and mobile-first - the design needs to adjust to the available space (different screen sizes)

Instructions

  • Visit Denkmal.org and have a look at what's already there.
  • Think about what elements and functionalities need to be included
  • Make at least two drafts (One for the integration on the landing page, one for the radio interface)

We hope you have fun working on this mini-project and are looking forward to examine your solution.
